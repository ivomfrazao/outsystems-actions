Everything you need to stay on top of deployments

🔔

Real-time notifications

Get browser notifications the moment a deployment finishes — success, warning, error, or intervention required. Configure exactly which outcomes you care about.

📋

Deployment history

Keep track of recent deployments with a configurable history — set a max count or a max age in days. Click any card to jump straight to the deployment page.

🔵

Tab status indicators

Active deployment tabs get an emoji prefix in the title so you can see the status at a glance across all your open tabs without switching to the extension popup.

🌙

Dark mode support

Follows your system preference or lets you override — choose Light, System, or Dark in the extension settings. Every element adapts cleanly.

🔍

Stale deployment detection

Closed your browser mid-deploy? When you reopen it, the extension marks unresolved deployments as Unknown. Reopen the page to resolve them to their real status.

⚙️

Flexible configuration

Toggle notifications per outcome, set history limits, enable or disable card animations. A master notification toggle lets you silence everything with one click.

Works where you work

SC

Service Center

Monitors eSpace and Solution publish operations. Shows app name, environment, and outcome. Distinguishes Deploy from Solution publishes in the card meta row.

LT

LifeTime

Monitors deployment plan executions. Extracts deployed application names from the plan (e.g. MyApp +2) so you always know what was deployed and where.

Get started in seconds

From the store

Install directly from the browser extension store — no build step required.

Edge Add-ons
or

From source

  1. Clone the repo and run npm ci && npm run build
  2. Go to chrome://extensions/ (or edge://extensions/)
  3. Enable Developer mode
  4. Click Load unpacked and select the dist/ folder

Permissions used

The extension requests only the permissions it needs to function:

notifications Show browser notifications when deployments complete
storage Persist deployment history and your preferences locally
tabs Update the tab title with status indicators; reuse existing tabs when opening a deployment
scripting Inject the content script that reads deployment status from the page
host: */ServiceCenter/* and */LifeTime/* Access your OutSystems environments — no data is sent externally
